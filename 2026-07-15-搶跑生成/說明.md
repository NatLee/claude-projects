# 搶跑生成 · 小模型打草稿、大模型驗收

**類別**：科學趣聞
**形式**：speculative decoding（推測解碼）看得見的動畫 ＋ 並排計速對照
**日期**：2026-07-15

---

## 這是什麼

大語言模型生成文字時，一次只能吐一個字（自迴歸），所以很慢。**推測解碼（speculative decoding）**是一招神奇的加速法：

1. 讓一個**便宜的小模型**先一口氣猜好幾個字（草稿）。
2. 讓**貴的大模型一次平行驗收**這一整串草稿（不是一個一個，是一次全部）。
3. **接受**前面連續猜對的字，在**第一個猜錯**的地方由大模型**修正**，再從那裡繼續下一輪。

最反直覺的地方是：**最後產生的文字，和大模型自己一個一個慢慢寫出來的一模一樣——卻快了好幾倍。** 這頁把這整個過程做成動畫，並把兩種做法並排賽跑給你看。

## 怎麼玩

- **設定**：選一份語料（唐詩選／三字經／論語），拉三根滑桿：
  - **草稿長度 γ**：小模型一次猜幾個字（1–8）。
  - **小模型聰明度**：越高，草稿越常猜中大模型的選擇（0–100%）。
  - **小模型成本比 c**：草稿一步相對於大模型一次的成本（用來估「時間加速」）。
- **看一輪推測解碼**：按「跑一輪」，看小模型吐草稿 → 大模型驗收 → 綠色接受／紅色拒絕並修正。或按「自動播放」連續跑。
- **哇時刻·計速對照**：按「開始賽跑」，左邊「大模型逐字」每次呼叫只前進一個字，右邊「推測解碼」每次呼叫前進一整輪。兩邊文字完全相同，但右邊用的呼叫次數少很多。
- **甜蜜點長條圖**：畫出「估計時間加速」如何隨 γ 變化——猜太多但常錯反而浪費草稿算力，所以 γ 有個最佳值。
- 鍵盤：所有按鈕與滑桿都可用 Tab 聚焦、Enter／空白操作；長條圖每根可聚焦並用 Enter 切換 γ。

## 接受／拒絕規則與無損保證（重點）

加速卻不改變結果，靠的是一條保證「輸出分布不變」的接受規則。這是**精確、無損**的，**不是近似**。

**一般規則（取樣）**：設某位置大模型機率為 `p`、小模型為 `q`，小模型抽到草稿字 `x`：

```
接受機率 = min(1, p(x) / q(x))
若拒絕，改從修正後的分布重抽： p'(x) ∝ max(0, p(x) − q(x))
```

可以證明：這樣產生的字，**邊際分布恰好等於直接用大模型 `p` 取樣**。所以整段輸出在統計上與大模型單獨解碼完全等價——無損。

**本頁動畫用的是它在溫度→0（貪婪）時的精確特例**，化簡成很直覺的一句：

```
草稿字 == 大模型的選擇 ？ 接受 : 拒絕 → 改成大模型的選擇
```

於是每個吐出來的字，都等於大模型在該位置本來就會選的字，**因此和大模型逐字生成逐字相同**。這也是本頁能做到「兩邊輸出一字不差」的原因。

## 加速從哪來

關鍵觀察：**平行對一小段草稿「評分」的延遲，和從大模型抽一個字差不多。** 所以：

- 大模型驗收 γ+1 個位置只需**一次**平行前向傳遞（1 次呼叫）。
- 這一次呼叫，若接受了 k 個草稿，就一口氣前進 **k+1** 個字（k 個接受 ＋ 1 個修正或 bonus）。
- 逐字解碼要 N 次呼叫才生 N 個字；推測解碼只要「輪數」次呼叫。**加速 = 字數 ÷ 輪數。**

加速倍數取決於兩件事：小模型的**猜中率 α**，與大小模型的**成本比**。猜中率越高、成本比越小，加速越大；但 γ 太大而 α 不夠高時，草稿算力被浪費，所以有個**甜蜜點**。

## 這是玩具模型（誠實聲明）

本頁的「大模型」「小模型」是兩個 **n-gram 下一字預測器**（大模型高階、近乎確定，作為 ground truth；小模型低階、便宜、偶爾猜錯），純粹用來把演算法演出來，**完全不呼叫任何 LLM 或 AI API**。「小模型聰明度」是一個讓你探索的旋鈕，用決定性方式調整草稿的猜中率。

真實系統的 draft／target 是神經網路，但**接受／拒絕規則、無損保證、以及加速的來源（草稿平行驗收）都與這裡完全一致**。這頁示範的是演算法骨架，不是效能數字。

## 驗證

核心演算法寫成環境無關的純函式，用 node 測試腳本（`spec-test.js`）驗證，共 **1036 項斷言全過**：

- **無損性**：189 組設定（3 語料 × 9 聰明度 × 7 種 γ）、共 6,426 個 token，speculative 輸出與大模型單獨貪婪解碼**逐 token 完全相同**。
- **加速**：大模型呼叫次數確實更少；γ=4 時加速隨聰明度落在 **×1.03 ～ ×4.86**。預設聰明度下：唐詩選 ×3.40（α=77%）、論語 ×2.43（α=66%）、三字經 ×2.13（α=57%）——與論文回報的 2–3× 同一量級。
- **接受／拒絕規則**：200 輪中 174 輪發生拒絕修正、26 輪全接受＋bonus；每次拒絕的修正字都獨立重算並確認等於大模型該位置的選擇。
- **隨機規則的分布無損**（bonus）：用攤開的合成分布，40 萬次取樣，`min(1,p/q)`＋residual 重採產生的經驗分布與大模型 `p` 的最大機率差 **max|Δ| = 0.0017**。
- **甜蜜點**：α=0.7、c=0.15 時，估計時間加速隨 γ 先升後降，最佳 γ=3。
- **安全**：無 fetch／XHR／AI API、無外部資源、localStorage 一律 `spec.` 前綴。

## 技術

- 純靜態三檔＋說明，**零外部資源**、離線可直接雙擊開啟、**無任何 fetch，不呼叫任何 AI**。
- 核心（亂數、n-gram 模型、接受／拒絕解碼器）是環境無關的純函式，瀏覽器與 node 測試共用同一份程式。
- 動畫只用 transform／opacity；賽跑用 rAF 節流，並在**分頁隱藏或捲出畫面時自動暫停**。
- `prefers-reduced-motion` 完整降級（直接顯示結果、少動畫；以 matchMedia change 動態監聽）。
- 設定存在本機 localStorage，key 一律 `spec.` 前綴：`spec.corpus`、`spec.gamma`、`spec.kappa`、`spec.cost`。
