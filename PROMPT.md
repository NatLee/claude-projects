# 每日子頁面 Prompt

> 給每日自動化排程使用的 prompt。整個工作資料夾是一個「大專案」：一個純靜態網站，
> 入口是根目錄的 `index.html`，每天為它長出一個新的子頁面。

---

目標：每天早上自動為使用者的「每日小專案」靜態網站（位於 C:\Users\NatLee\Desktop\claude-projects，已是 git 倉庫，入口為根目錄 `index.html`）新增一個全新、好玩、有質感的**子頁面**，帶來驚喜。全程使用繁體中文。

【大專案架構】整個資料夾是一個可離線瀏覽、可直接部署（如 GitHub Pages）的純靜態網站：

- 根目錄 `index.html`：首頁，卡片式目錄。專案清單寫在其中一個標示清楚的 `const PROJECTS = [...]` 陣列裡，卡片由陣列自動生成。
- 每個子頁面一個資料夾：`YYYY-MM-DD-簡短專案名/`，入口一律是該資料夾內的 `index.html`，並附一份 `說明.md`。
- 全站禁止任何後端與資料庫：需要儲存資料時一律使用 localStorage。

【主題類別】每天從以下四大類「隨機」挑選一類（盡量輪流、避免連續重複）：

1. 網路趣聞・冷知識
2. 生活痛點小工具
3. 學習新知
4. 創意・娛樂

【產出形式】一律是純靜態子頁面（HTML/CSS/JS，可直接用瀏覽器離線打開），但**風格與互動形式每天都要有變化**：

- 可以是精緻的多檔頁面（index.html + style.css + app.js），也可以是單檔自包含頁面。
- 可從 cdnjs 載入函式庫；除此之外不得依賴任何外部資源（離線打開必須完整可用）。
- 需要儲存使用者資料（紀錄、收集進度、統計）時使用 localStorage，且所有 key 必須加上該專案專屬的英文短前綴（例如 `gacha.`、`fermi.`），因為全站部署在同一網域下，避免不同子頁面互相覆蓋。
- 互動形式要輪替：遊戲、模擬器、視覺化、工具、收集系統、文字冒險、聲音玩具……不要連續幾天都同一型。

【執行步驟】

1. 避免重複：先列出根目錄下既有的 `YYYY-MM-DD-*` 資料夾並瀏覽 `README.md` 目錄，看看最近做過哪些「類別」與「互動形式」。這次刻意挑一個最近沒做過的類別與形式、全新的題材。

2. 構思點子：挑一類主題，想一個具體、有趣、做得完的點子。若屬於「網路趣聞」或「學習新知」，先用 WebSearch 找一兩個真實素材或數據當靈感，確保內容正確、有料。

3. 建立子頁面：在根目錄建立資料夾 `YYYY-MM-DD-簡短專案名`（日期用今天，台灣時區），裡面放 `index.html`（入口）與其他需要的檔案。介面與文字全部使用繁體中文，設計要精緻、有巧思、避免陽春樣板感。頁面左上角（或其他不干擾的位置）放一個低調的「← 回首頁」連結指向 `../index.html`，讓子頁面與全站連成一體。

4. 測試（很重要）：
   - 抽出所有 JS（含內嵌 `<script>`）跑 `node --check` 確認語法無誤；或用無頭瀏覽器確認可載入、無主控台錯誤。
   - 確認頁面直接雙擊（file://）開啟可完整運作：相對路徑正確、無 fetch 後端呼叫、外部資源只來自 cdnjs。
   - 若有 localStorage，確認 key 都有專案前綴。
   修正所有明顯問題，確保交付的是「能跑」的成品。

5. 附上說明：在資料夾放一個 `說明.md`，介紹這是什麼、怎麼玩／用；說明為什麼有趣或實用；資料來源（若涉及事實）；並註明資料儲存在瀏覽器 localStorage（若有用到）。

6. 掛上首頁：編輯根目錄 `index.html` 的 `PROJECTS` 陣列，把今天的專案**加在陣列最上方**，欄位照既有格式：`date`、`title`、`emoji`、`dir`（資料夾名）、`category`（四大類之一，字串需完全一致）、`desc`（一句話介紹）。不要動陣列以外的程式碼。

7. 更新目錄：在根目錄 `README.md` 的專案紀錄區最上方，加上今天這個專案的一行紀錄（日期、名稱、類別、一句話簡介，連結指向該資料夾的 index.html）。

8. 提交 git：以繁體中文 commit message 提交今天的變更，格式如「新增子頁面：YYYY-MM-DD 專案名（類別）」。只 commit、不 push。

9. 呈現給使用者：用 present_files 秀出今天子頁面的 `index.html`，並用一兩句話介紹今天的驚喜。

【風格與原則】

- 全程繁體中文。
- 每天都要「不一樣」：主題、風格、互動方式都要有變化，但都活在同一個靜態網站裡。
- 小而完整：範圍小到能在一天內做完並測過，但要做到能用、沒有明顯 bug。
- 真實正確：凡涉及事實或數據，務必正確（必要時用 WebSearch 查證）。
- 全站一致的底線：純靜態、離線可用、localStorage 加前綴、掛上首頁與 README、回首頁連結——其餘的美感與玩法盡情發揮。
- 重點是趣味、美感與實用，讓使用者每天早上打開首頁時眼睛一亮。
