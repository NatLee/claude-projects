# 每日小專案 📦

這是一個**純靜態網站**：每天早上 6:00 自動長出一個全新的互動子頁面。
主題從六大類隨機抽選：網路趣聞・冷知識、奇聞軼事、科學趣聞、學習新知、生活痛點小工具、創意・娛樂。

👉 **入口**：直接用瀏覽器打開根目錄的 [index.html](./index.html)。

每個子頁面一個資料夾（各附 `說明.md`），入口都是資料夾內的 `index.html`，不需任何後端；需要儲存的資料一律存在瀏覽器 localStorage（清除瀏覽資料會一併清除紀錄）。每日產出流程見 [PROMPT.md](./PROMPT.md)。

- **2026-07-14** · [草莓的 R · 為什麼 AI 數不出草莓有幾個 r](./2026-07-14-草莓的R/index.html) 〔學習新知 · 瀏覽器內真訓練 BPE tokenizer localStorage〕— 你問 AI「strawberry 有幾個 r」，它常常數錯——不是笨，是它**看不到字母**。語言模型的輸入是 token，不是字元。這頁讓你在瀏覽器裡**當場訓練一個真的 BPE（byte-pair encoding）tokenizer**：選內建語料或貼自己的文字，按「訓練」就看它一輪一輪把最高頻的相鄰 pair 合併（`e ＋ r → er · 143 次`），可調合併次數（詞表大小）；再把任意文字即時切成彩色 token 色塊、標上 id。**哇時刻是「數 r」小遊戲**：左邊用字元數出正解（3 個 r），右邊顯示模型看到的其實是 `[st][raw][berry]`——它連字母都沒有，怎麼數？外加「同一句話中文 vs 英文的 token 數對照」，親眼看中文在英文詞表下有多吃虧。BPE 原是 **Philip Gage 1994**《C Users Journal》的**資料壓縮**演算法，2016 由 **Sennrich, Haddow & Birch**（ACL, pp.1715–1725）帶進 NLP 解未登錄詞；GPT-2/3 詞表 **50,257**，GPT-4 `cl100k_base`（~10 萬）、GPT-4o `o200k_base`（~20 萬）。含 **glitch token** 解說（` SolidGoldMagikarp`、` petertodd` 等，Rumbelow & Watkins 2023 發現，多半是 Reddit r/counting 使用者名稱這類「tokenizer 訓練到、模型沒訓練到」的殘留），並誠實標註「9.11 > 9.9」只是部分肇因。以 node **28 項測試**驗證（合併次數正確、encode 串回去無損含中文與 emoji、詞表隨合併遞增、strawberry 預設切成 3 個 token）。純靜態三檔、零外部資源、離線可用；localStorage 加 `tok.` 前綴。（來源：Gage 1994、Sennrich et al. 2016 ACL、Rumbelow & Watkins 2023 SolidGoldMagikarp）
