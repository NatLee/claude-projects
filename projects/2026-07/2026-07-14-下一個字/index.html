下一個字

ChatGPT 在做的事，本質上只有一件：猜下一個字。 這頁讓你在瀏覽器裡當場訓練一個真的語言模型——不是神經網路，是它的祖先， 一個只會數次數的 n-gram。它一個字都不懂，卻和 GPT 用著同一個目標函數。

1餵它一些字

它只知道你餵給它的東西。換一份語料，它就換一個腦袋。

也可以貼上你自己的文字：一封信、一篇日記、一份會議紀錄——它會學會那個人的口氣。

    3三個你會親眼看到的現象

    把滑桿推到極端，它會露出馬腳——而那三個馬腳，大型語言模型至今都還在對付。

    • 復讀機 把溫度調到 0，看它會發生什麼事。
    • 胡言亂語 把溫度調到 1.6 以上。
    • 它在背書 把 n 調到 5，讓它多寫幾十個字。

    它跟 GPT 的距離，比你以為的近

    1948

    Shannon 早就玩過這個

    夏農在〈通訊的數學理論〉裡，就用 n-gram 手工生成過近似英文：他真的是翻著一本書，隨機挑字、往前翻頁找下一個接續，一路做出零階、一階、二階、三階近似——階數越高，越像英文。

    他同時算出：不考慮八個字母以外的結構，英文大約有 50% 的冗餘——你寫下的每一個字，有一半是被語言的規則決定的，只有另一半是你自己選的。

    同一個目標

    GPT 也只是在猜下一個 token

    自迴歸語言模型的訓練目標就是 next-token prediction：給前文，最大化下一個 token 的機率。這頁的 n-gram 用「數次數」估這個機率，Transformer 用神經網路把上下文壓縮成向量再估——目標函數一模一樣，差的是怎麼記住上下文。

    所以你在 API 上看到的 temperature、top_k、top_p，這裡的滑桿就是同一回事。

    n → 5

    為什麼 n-gram 撐不到 GPT

    n 每加一，可能的上下文就乘上一個字表大小。很快地，幾乎每個上下文在語料裡都只出現過一次——這時候「機率」只剩一個候選，模型別無選擇，只好把原文一字不差地背出來。

    這就是資料稀疏：n-gram 沒有「相似上下文」的概念。而 Transformer 把字變成向量，語意相近的上下文自然靠在一起——它才能在沒見過的句子上還說得出話。

    2^H

    困惑度：它在幾個字之間猶豫

    把分布的熵 H 取 2 的次方，就得到「有效候選數」。它等於 1，代表模型已經確定；等於 40，代表它心裡有 40 個差不多好的選項。這個數字就是困惑度（perplexity），也是評比語言模型最常用的指標——越低，代表它越不迷惘。

    溫度會直接改變它：溫度趨近 0，有效候選數被壓成 1（於是開始復讀）；溫度拉高，分布被攤平，它開始亂猜。

    純靜態、零外部資源、離線可用。所有統計與生成都在你的瀏覽器裡當場計算，語料不會離開這台電腦。
