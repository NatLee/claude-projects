# 從雜訊生形

在瀏覽器裡**當場訓練一個真正的擴散模型**（2D 點雲版），再親眼看一團**純雜訊**被反向去噪、
一步步收斂成你選的形狀。這正是 DALL·E、Stable Diffusion、Midjourney 背後那個「把雜訊擦掉」的引擎，只是縮到二維。

## 這是什麼

擴散模型（diffusion model）是目前文字生圖的主流做法。它的核心觀念反直覺得漂亮：

- **前向過程**：拿一筆真實資料，分 T 步慢慢摻入高斯雜訊，直到它變成一團看不出結構的純雜訊。
- **反向過程**：訓練一個網路，看著任何「半糊」的資料就能猜出裡面被加了哪一團雜訊；
  把它扣掉、往回走一步，重複 T 次，就能從純雜訊裡「長」出一個像資料的新樣本。

這一頁把資料換成 **2D 點雲**（一堆平面上的點所組成的形狀），
其餘全部照擴散模型的真章來做：加噪排程、預測雜訊 ε、反向採樣，一步都沒省。

## 怎麼玩

1. **選一個形狀**：愛心、圓環、雙月、五角星，或選「自己畫」用滑鼠／手指塗一團點。
2. **調設定**（可選）：
   - **雜訊步數 T**（30 / 50 / 70）：把資料揉成雜訊、再還原所用的步數。
   - **訓練程度**（快 / 標準 / 精細）：訓練幾步，換取速度與品質。
   - 改了形狀、T 或訓練程度，都需要**重新訓練**（畫面會提示）。
3. **開始訓練**：看損失曲線往下掉，數秒內收斂。這是真的在做反向傳播。
4. **撒一團雜訊，開始生成**：
   - 右邊（**反向**）從純雜訊出發，一步步去噪，收斂成你的形狀——這就是「哇時刻」。
   - 左邊（**前向**）同時把你的形狀加噪、溶解成雜訊，兩邊時間軸相反、互相對照。
   - **採樣速度**可調（慢可以看清楚單步）；也能**單步去噪**、**拖曳時間軸**、**重播**。

全部即時運算，沒有任何預先錄好的動畫。

## 為什麼有趣（跟文字生圖的關係）

真實的文字生圖，跟這一頁**演算法骨架一模一樣**，差別只在於規模與條件：

| | 這一頁（玩具版） | 真實圖像擴散 |
|---|---|---|
| 資料 | 2 維的點 | 幾十萬～幾百萬維的像素 |
| 網路 | 小型 MLP（幾千參數） | U-Net（帶注意力，上億參數） |
| 條件 | 無 | **文字**（`ε_θ(x, t, 文字)`） |
| 空間 | 直接在資料上 | 常在壓縮後的**潛在空間**（latent diffusion） |

- **文字條件**：把你打的字編碼後餵進網路，去噪時就會「照著字」把雜訊推向對應的圖。
- **classifier-free guidance**（Ho & Salimans 2022）：同時算「有文字」與「沒文字」兩種預測，
  往有文字那邊多推一點，讓成品更貼題。
- **latent diffusion**（Rombach 等人 2022，即 Stable Diffusion）：先把圖片壓進小很多的潛在空間，
  在那裡做擴散，最後才解碼回像素——這是它能在消費級顯卡上跑的關鍵。

但無論加了多少東西，**加噪排程、預測 ε、反向採樣**這三件事都沒變。看懂這一頁，就看懂了引擎。

## 演算法在做什麼

- **前向加噪**（封閉式，不用學）：`x_t = √ᾱ_t · x₀ + √(1−ᾱ_t) · ε`，其中 `ε ~ N(0, I)`。
- **排程**：本頁用 **cosine schedule**（Nichol & Dhariwal 2021），由 `ᾱ_t` 反推 `β_t`，
  對任何 T 都能把 `ᾱ_T` 帶到接近 0（也就是最後真的變成純雜訊）。
- **訓練目標**（DDPM 的簡化損失）：`L = ‖ε − ε_θ(x_t, t)‖²`——一個普通的均方誤差，
  網路 `ε_θ` 輸入是 `(x, y, t)`（t 用傅立葉時間嵌入），輸出是預測的雜訊 `(εx, εy)`。
- **反向採樣**（DDPM ancestral sampling）：從 `x_T ~ N(0, I)` 出發，
  `x_{t−1} = 1/√α_t · (x_t − β_t/√(1−ᾱ_t) · ε_θ) + σ_t · z`，一路走到 `x₀`。
- **實作細節**：SiLU 激活（比 tanh 穩定很多，tanh 版在高雜訊會外插爆炸、採樣直接發散）、
  Adam＋cosine 學習率衰減、權重 EMA（讓採樣乾淨、少離群點）。

核心程式（`app.js` 裡的 CORE 區段）不碰 DOM，可以在 node 裡直接 `require` 進來測試。
實測結果：反向傳播的解析梯度與數值梯度**最大相對誤差只有 9×10⁻⁹**；
訓練 2800 步後，各形狀「生成點雲到目標的平均最近鄰距離」明顯小於「純雜訊到目標」的距離
（圓環約 1/8，雙月約 1/3，愛心、五角星約 1/2）——它是真的把雜訊推到形狀上。

## 資料來源

- [Sohl-Dickstein, Weiss, Maheswaranathan & Ganguli (2015) — Deep Unsupervised Learning using Nonequilibrium Thermodynamics (arXiv:1503.03585)](https://arxiv.org/abs/1503.03585)：最早提出擴散式生成模型。
- [Ho, Jain & Abbeel (2020) — Denoising Diffusion Probabilistic Models / DDPM (arXiv:2006.11239)](https://arxiv.org/abs/2006.11239)：把訓練目標簡化成「預測雜訊 ε」，讓擴散真正好用。
- [Nichol & Dhariwal (2021) — Improved Denoising Diffusion Probabilistic Models (arXiv:2102.09672)](https://arxiv.org/abs/2102.09672)：本頁採用的 cosine 雜訊排程。
- [Rombach, Blattmann, Lorenz, Esser & Ommer (2022) — High-Resolution Image Synthesis with Latent Diffusion Models (CVPR 2022, arXiv:2112.10752)](https://arxiv.org/abs/2112.10752)：Stable Diffusion 的潛在空間擴散。
- [Ho & Salimans (2022) — Classifier-Free Diffusion Guidance (arXiv:2207.12598)](https://arxiv.org/abs/2207.12598)：文字條件生成常用的引導技巧。

## localStorage 說明

本頁只用 localStorage 記住你的介面偏好，全部加上 `diff.` 前綴，不含任何個資：

- `diff.shape`：上次選的形狀（heart / ring / moons / star / draw）。
- `diff.T`：上次選的雜訊步數 T。
- `diff.effort`：上次選的訓練程度（fast / std / fine）。
- `diff.speed`：上次選的採樣速度（slow / normal / fast）。
- `diff.runCount`：你按過幾次「生成」（純粹是個小計數器）。

全部在你的瀏覽器裡跑，沒有任何網路請求、沒有外部模型檔，雙擊 `index.html` 即可離線運作。
