# 騙倒它

**你先親手訓練一個真的神經網路分類器，然後親手用梯度把它騙倒。**

加上一層人眼幾乎看不出來的雜訊，它就會信心滿滿地答錯 —— 而且答得比原本還篤定。

---

## 這是什麼

一個會真的算梯度的實驗場。頁面上沒有任何預先寫死的結果、沒有錄好的動畫、沒有外部模型檔：

- **資料**是程式當場生成的：3,000 張 16×16 灰階小圖，三類形狀（圓／方／三角），隨機位置、大小、旋轉、前景背景亮度、高斯雜訊。三類的**面積刻意設成一樣**，這樣模型沒辦法靠「哪一類比較亮」偷吃步，只能真的去看形狀。
- **模型**是純 JS 手寫的多層感知器：256 → 64 → 3，16,643 個參數，ReLU + softmax + 交叉熵，Adam 最佳化器。反向傳播是自己算的。
- **攻擊**是課本上那條 FGSM 公式，一字不差。

## 怎麼玩

1. **訓練**：按「開始訓練」。幾秒內你會看到 loss 曲線往下掉、測試準確率往上爬到 97~99%。訓練分批塞進 `requestAnimationFrame`，所以 UI 不會凍住；切到別的分頁或把卡片捲出畫面，它會自動暫停。
2. **攻擊**：模型會挑一張它答對、而且信心 >90% 的圖。拉動 **ε 滑桿**，畫面即時算出對抗樣本。並排三張圖：原圖 ／ 放大過的雜訊 ／ 對抗圖。看下面兩排信心條 —— 左邊「圓形 99%」，右邊「三角形 96%」，而那兩張圖你根本看不出差別。
   - 按「**自動把 ε 拉大，直到它翻掉**」，看它在哪一刻崩潰。
   - **目標攻擊**：指定你要它答成哪一類，用 40 步的 PGD 逼它就範。
3. **換你出手**：在小畫布上自己塗一個形狀，先看它怎麼分類，再對你畫的圖算梯度、攻擊它。

## 演算法在做什麼

### FGSM（Fast Gradient Sign Method）

```
x_adv = clip( x + ε · sign( ∇ₓ J(θ, x, y) ) )
```

`∇ₓ J` 是「損失函數對**輸入像素**的梯度」—— 不是對權重，是對圖片本身。它指出「往哪個方向改這張圖，模型會錯得最快」。
取 `sign` 只保留方向、丟掉大小，然後每個像素都往那個方向推 ε 這麼多。一次前向、一次反向，就結束了。

### 目標式 PGD（Projected Gradient Descent）

```
重複 k 次：
    x ← x − α · sign( ∇ₓ J(θ, x, y_target) )      # 讓「目標類別」的損失變小
    x ← clip( x, x₀ − ε, x₀ + ε )                 # 投影回 x₀ 的 L∞ ε 球
    x ← clip( x, 0, 1 )                           # 還要是合法的像素值
```

多走幾小步、每步都拉回 ε 預算內，就能不只讓它答錯，而是**指定它答成哪一類**。

### 為什麼這樣就能騙倒它

反直覺的答案：**不是因為模型太複雜，而是因為它太線性。**

線性單元的輸出是 `w·x`。輸入加上擾動 η，輸出就多了 `w·η`。若取 `η = ε·sign(w)`，這一項等於 `ε · Σ|wᵢ|` —— **會隨維度數量線性增長**。
單一像素只動 0.7%，人眼看不出來；但幾十萬個像素同時朝著對它最壞的方向動 0.7%，加起來就足以跨過決策邊界。ReLU 網路在局部幾乎是分段線性的，所以照樣中招。

這也解釋了對抗樣本的**可遷移性**：不同模型學到的線性方向很像，所以騙倒 A 的圖常常也騙得倒 B。

## 實測數字（用 node 跑同一份核心程式碼）

| 項目 | 結果 |
|---|---|
| 測試準確率（900 張沒看過的圖） | **97.8%**（5 個不同亂數種子：98.4 / 99.1 / 99.1 / 99.3 / 99.4%） |
| 梯度檢查（中央差分 vs 反向傳播） | 最大相對誤差 **2.4×10⁻⁶**（含對輸入的梯度 dx） |
| FGSM 翻轉率（頁面預設 ε = 0.08 ≈ 20/255） | **97.6%**，翻轉後平均信心 **93.2%** |

| 可遷移性（A 的對抗樣本騙倒獨立訓練的 B） | ε=0.08 → **77.6%**；ε=0.12 → **90.2%** |

## 誠實說明

這是一個**小模型 + 合成資料**的示範，不是 ImageNet。但演算法（FGSM／PGD／交叉熵／反向傳播）與論文中的一模一樣，畫面上每個百分比都是你的瀏覽器現場算的。

有一點要講清楚：這裡只有 256 個像素，雜訊沒什麼地方可躲，所以 ε 一大你就會看到顆粒感。真實照片有幾十萬個像素，**同樣強度的擾動反而更難察覺** —— 這正是「高維度讓小改動的槓桿變長」的另一面。換句話說，真實世界的模型不是比較安全，是**比較好騙**。

## 資料來源

- [Szegedy et al. (2013) — *Intriguing properties of neural networks*, arXiv:1312.6199](https://arxiv.org/abs/1312.6199)：第一次描述對抗樣本（當時稱為神經網路的「盲點」），並指出它們會在不同模型間遷移。
- [Goodfellow, Shlens & Szegedy (2014) — *Explaining and Harnessing Adversarial Examples*, arXiv:1412.6572](https://arxiv.org/abs/1412.6572)：提出 FGSM。Figure 1：熊貓（信心 **57.7%**）+ **.007** × sign(∇ₓJ) = 長臂猿（信心 **99.3%**）；中間那張雜訊圖本身被 GoogLeNet 判為「線蟲」，信心 8.2%。核心論點：對抗樣本源自模型的**線性**，不是非線性或過擬合。
- [Brown et al. (2017) — *Adversarial Patch*, arXiv:1712.09665](https://arxiv.org/abs/1712.09665)：一張可以印出來的迷幻貼紙放在香蕉旁邊，VGG16 從「香蕉 97%」變成「烤麵包機 99%」。
- [Eykholt et al. (2018) — *Robust Physical-World Attacks on Deep Learning Visual Classification*, CVPR](https://openaccess.thecvf.com/content_cvpr_2018/papers/Eykholt_Robust_Physical-World_Attacks_CVPR_2018_paper.pdf)：在真的停車標誌上貼幾張黑白貼紙 → 讀成「速限 45」。實驗室拍攝的影像 **100%** 誤判，行進車輛拍的影片幀 **84.8%** 誤判。
- [OpenAI (2021) — *Multimodal Neurons in Artificial Neural Networks*](https://openai.com/index/multimodal-neurons/)：CLIP 的印刷體攻擊 —— 在蘋果上貼一張手寫「iPod」的紙條，CLIP 就說那是 iPod。所需裝備：一支筆、一張紙。
- [Elsayed et al. (2018) — *Adversarial Examples that Fool both Computer Vision and Time-Limited Humans*, NeurIPS](https://papers.nips.cc/paper/7647-adversarial-examples-that-fool-both-computer-vision-and-time-limited-humans)：能跨模型遷移的對抗樣本，在觀看時間受限（約 2 秒）的情況下，也會把**人類**的判斷推向攻擊者指定的類別。
- [Madry et al. (2017) — *Towards Deep Learning Models Resistant to Adversarial Attacks*, arXiv:1706.06083](https://arxiv.org/abs/1706.06083)：PGD 攻擊與對抗訓練（adversarial training，目前最主流的防禦）。

## 技術細節

- **純靜態、零外部資源**：不引用任何 CDN、模型檔或資料集，沒有任何 `fetch`。雙擊 `index.html` 就能離線跑。
- **效能**：訓練分批放進 `requestAnimationFrame`；分頁隱藏（`visibilitychange`）或卡片離開視窗（`IntersectionObserver`）時自動暫停。
- **動畫紀律**：只用 `transform` / `opacity`；完整支援 `prefers-reduced-motion`（並動態監聽 `matchMedia` 的變化）。
- **無障礙**：鍵盤可完整操作、有聚焦樣式、動態結果用 `aria-live` 播報；不方便用滑鼠畫圖的話，第三步提供「放一個圓／方／三角」按鈕。

### localStorage

全部以 `advx.` 為前綴，只存三個小東西，沒有任何個資：

| Key | 內容 |
|---|---|
| `advx.eps` | 你上次用的雜訊強度 ε |
| `advx.foolCount` | 你總共騙倒它幾次 |
| `advx.bestAcc` | 你訓練出來的最佳測試準確率 |

清掉瀏覽器資料就會全部消失。
