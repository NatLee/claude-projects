專家會診

兆級參數，每次只醒來一小撮

很多人以為，大型語言模型每吐一個字，都得動用它全部的參數。其實像 Mixtral、DeepSeek、以及被廣泛傳聞的 GPT-4 這類採用「專家混合（Mixture of Experts, MoE）」的模型，內部養了一大群「專家」子網路， 但每個字只會被一個「路由器」送去其中少少幾個——所以它能有上兆參數， 每個字卻只叫醒一小撮。這頁把「稀疏啟動 ＋ 路由」變成看得見、能親手玩的東西， 而且底下真的在跑一個小型 MoE：路由、稀疏加總、負載平衡都是當場算的。

01

路由台：看每個字被送去哪幾個專家

打一段字（中文以「字」為一個 token），或選一句範例。每個 token 通過路由器， 亮起它被送去的 k 個專家，其他專家保持暗——暗的就是沒被啟動、不花計算。 這裡的路由與稀疏加總都是當場算的，不是預錄動畫。

中文一個字＝一個 token；也可打英數字。

top-k
路由器 ·

gating：對每個專家算分數 → 取 top-k

按「整句送入」或「下一個字」，看每個 token 被路由到哪些專家。

—這個 token 用了幾個專家
—這個 token 的專家啟動比例
0整句用到的不同專家
0已處理的 token 數

專家的顏色只是編號識別；路由器一定要算（它負責決定去哪），真正稀疏、可以省下的是專家。 同一個字每次都走同一條路（決定性）；不同字走哪裡，是這個小 MoE 訓練後學出來的。

02

哇：總參數很大，這個 token 只醒來一小撮

MoE 最反直覺的地方——參數量決定它「學得下多少東西」， 每個 token 實際動用的參數才決定「這一步要算多少」。兩者可以差很多。挑一個模型看看。

— 總參數
— 這個 token 只醒來
— 啟動比例
總參數（容量） 本 token 啟動（算力）

數字都是查證過的公開資料；GPT-4 的部分是非官方傳聞（OpenAI 從未證實），已明確標示。 Switch-C 用 top-1（每 token 只 1 個專家），所以強調的是「總參數 vs 每步算力」而非啟動比例。

03

負載平衡：別讓少數專家過勞、其他餓死

如果大家都愛去同一個專家，它會被操爆、其他專家卻閒著（甚至餓死、幾乎沒被訓練到）。 這裡灌進一批「偏心」的 token，開關負載平衡，看分布怎麼從失衡變均勻。

0已灌入 token
—分布 entropy（越大越均勻，上限 3.00）
—負載變異數（越小越平均）
—最忙專家 / 平均

關掉平衡：偏心的輸入會把少數專家操爆。打開平衡：路由器會把後來的 token 擠去比較閒的專家—— 分布變均勻，代價是有些 token 被送去「非本行」的專家，輸出會差一點。這正是真實 MoE 要拿捏的取捨。

這裡示範的是 DeepSeek-V3 那種「無輔助損失」的推論期偏置：依各專家目前的負載動態調整分數。 另一條路是 Switch Transformer 的訓練期輔助損失（鼓勵平均使用）——兩者這頁都有（後者見底下說明與測試）。

04

老實說：這是教學版小 MoE

哪些是真的、哪些是為了看得懂而簡化的，講清楚。

這些是真的在跑

  • 路由器是一層線性 gating＋softmax，真的算分數、真的取 top-k。
  • 只加總被選中專家的輸出——沒被選的專家完全沒算（稀疏計算）。
  • 這個小 MoE 是用梯度下降真的訓練出來的，router 自己學會把某些字送去某些專家。
  • 負載平衡（推論期偏置、訓練期輔助損失）都是真的機制，效果可被量化、可被 node 驗證。

這些是教學簡化

  • 維度與專家數遠小於真實模型（這裡 8 個專家、6 維；真實模型動輒上百專家、上千維）。
  • token 的向量是合成的（真實模型的是前面好幾層算出來的隱藏狀態）。
  • 畫面上的「特徵族」是我們造出來、方便你看見分群的； 真實模型的專家分工是學出來的，通常無法對應人類主題—— 不是「這個管數學、那個管寫詩」，別被直覺騙了。
  • 真實 MoE 的專家輸出會再往上疊很多層；這裡只示範「路由 + 稀疏 + 平衡」這個核心。

一句話：容量看總參數，算力看啟動參數。MoE 讓你把參數養得很大（記得住很多東西）， 每一步卻只叫醒一小撮（算得起）。路由決定叫醒誰，負載平衡讓大家別偏食—— 這套機制，這頁的小 MoE 和真實的大模型是一致的。

05

資料來源

概念源流與模型數字的出處（可點）。

  • Shazeer et al. (2017)．Outrageously Large Neural Networks: The Sparsely-Gated Mixture-of-Experts Layer．arXiv:1701.06538——稀疏門控 MoE 的源頭：上千個專家、top-k 路由、輔助負載平衡損失。
  • Fedus, Zoph & Shazeer (2021)．Switch Transformers．arXiv:2101.03961——把路由簡化成 top-1，首度穩定訓練到兆級（Switch-C 約 1.6 兆參數）。
  • Jiang et al. (2024)．Mixtral of Experts．arXiv:2401.04088——8 個專家、每 token 選 2；總約 47B、每 token 只用約 13B。
  • DeepSeek-AI (2024)．DeepSeek-V3 Technical Report．arXiv:2412.19437——256 路由專家 + 1 共享，每 token 啟動 8+1；總 671B、每 token 只用 37B；無輔助損失負載平衡。
  • Wang et al. (2024)．Auxiliary-Loss-Free Load Balancing Strategy for MoE．arXiv:2408.15664——依負載動態調整偏置來平衡（本頁「負載平衡」開關的做法）。
  • GPT-4 採 MoE 為非官方傳聞（George Hotz 等，2023；OpenAI 未證實）：約 8 個 220B 專家、每 token 選 2、估總 ~1.76T。
