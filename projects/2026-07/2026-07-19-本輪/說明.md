# 傅立葉本輪 · 用旋轉的圓，重畫你剛剛畫的線

在畫布上畫一條線（或選一個內建圖形），一串**首尾相接、各自旋轉的圓**會自己疊起來，最外面那顆圓的筆尖拖出的軌跡，會慢慢變成你畫的那條線。這不是預先錄好的動畫效果——底下真的在對你的線做**離散傅立葉變換（DFT）**，每一顆圓的半徑、起始角度、轉速，全部是從你畫的點算出來的係數。

## 怎麼玩

1. 在畫布上用滑鼠拖曳、或手指觸控畫一條線（線或封閉圖形都可以），放開就會自動接上本輪動畫。
2. 也可以直接點右側「圖形」選單，選內建的**愛心／星星／音符／簽名**四種示範圖形。
3. 拖動「**圓的數量**」滑桿（1 → 150），看粗糙的圓形輪廓一路變精細，最後幾乎完全貼合原圖——這是整頁最好玩的部分，建議先看 3 顆圓的樣子，再一路拉到 150。
4. 「顯示原始線稿」可疊出半透明的原圖對照；「顯示旋轉的圓」可以把本輪本身關掉，只留下描出的軌跡。
5. 下方「係數排行榜」列出貢獻最大的前十個頻率——目前圓數用不到的排名會變淡，讓你看見「多轉出來的圓」到底補上了哪些細節。
6. 全程可鍵盤操作：Tab 到圖形按鈕用 Enter／空白鍵選取，滑桿用方向鍵、Home、End 調整。

## 為什麼有趣（哇時刻）

一堆看起來毫無意義、彼此打轉的圓圈，疊在一起卻精確畫出你剛剛手寫的線條——這正是**傅立葉級數的幾何意義**，第一次用眼睛直接看見它，而不是在課本裡看一串加總符號。把圓數從 3 拉到 100，你會看到：

- 3～8 顆圓：只剩下最粗略的輪廓，像個晃動的圈。
- 十幾到幾十顆：轉折、尖角開始浮現。
- 接近甚至等於取樣點數（150）：幾乎與原圖分毫不差，右上角的「誤差」讀數會逼近 0px。

「本輪」這個詞本身就是個彩蛋：它是中文天文學翻譯托勒密模型時造的詞，本輪繞著均輪轉，古人用來解釋行星為什麼會「逆行」；千百年後，同一種「圓疊圓」的幾何，被數學家借來畫任意曲線。更巧的是，見下方歷史故事——第一個把這套算法寫下來的人，用的正是字面意義上的天文應用。

## 方法論

- **取樣**：不論是自由畫的線、還是內建圖形，都先用等弧長重新取樣成固定 **N = 150** 個點（`resamplePath`：沿路徑總長度切成 150 等分，對每一段做線性插值）。內建圖形（愛心／星星／音符／簽名）用參數式公式或 Catmull-Rom 雲線先產生密集錨點，再套同一套等弧長取樣，確保封閉與開放路徑都能公平比較。
- **把 2D 點視為複數**：每個取樣點 (x, y) 視為 z = x + iy，這是本頁唯一需要的原料。
- **離散傅立葉變換（DFT，暴力版，O(N²)）**：

  ```
  X_k = 1/N · Σ_{n=0}^{N-1} z_n · e^(−i·2πkn/N)　　for k = 0..N-1
  ```

  頻率以「居中」方式改成有號整數（k ≤ N/2 時 freq=k，否則 freq=k−N），這樣連續時間 t 之間的插值才會走最短、最平滑的路（而不是每個高頻項都在兩個取樣點之間瘋狂多繞好幾圈）——這也是讓動畫在取樣點之間依然順滑的關鍵。
- **本輪重建（逆變換）**：把係數依 **振幅 |X_k| 由大到小排序**，依序疊成一條向量鏈：第 i 顆圓的半徑 = |X_k|、起始相位 = arg(X_k)、角速度 = 該係數的（有號）頻率；上一顆圓的箭頭尖端就是下一顆圓的圓心。只取排序後前 M 個係數重建，就是介面上「圓的數量」M；M = N 時等於精確的逆 DFT，保證能一字不差地重建取樣點。
- **即時誤差（Parseval 捷徑）**：畫面上「誤差」讀數不是每次都暴力重算，而是用 Parseval 定理的推論——只用前 M 個（依振幅排序）係數重建時，取樣點的均方根誤差恰好等於「被捨棄的係數」振幅平方和的平方根。因為係數已經照振幅排序，捨棄的都是「目前排名之後」那一截，所以這個和只會隨 M 增加而遞減，等於直接證明了「圓越多、誤差單調下降」。
- **複雜度**：頁面內文解釋了暴力 DFT 是 O(N²)（每個係數要做 N 次乘加，N 個係數就是 N² 次），FFT 靠「奇偶對半拆、拆完再併回去」的分治法把它壓到 O(N log₂N)，並附上 N = 64 到 1,048,576 的實際次數對照表。
- **無障礙與動態降級**：`prefers-reduced-motion` 開啟時，不做連續旋轉，改成一次算出整條軌跡（用 360 個取樣點沿 t = 0..1 描一次）直接畫出完整結果，並透過 `matchMedia(...).addEventListener("change", ...)` 動態監聽，使用者在頁面開著的狀態下切換系統設定也會立刻反映；rAF 動畫迴圈在分頁被切到背景（`document.hidden`）時會自動停止、回到前景才恢復。

## 測試與驗證結果

依規格做了三層測試，全部針對**實際出貨的 index.html** 內嵌 `<script>`（用正則抽出、跑 `node --check` 通過），而不是另外手寫一份相似的程式：

1. **DFT → 逆變換（用全部 150 個係數重建）能否還原原始取樣點**：對愛心／星星／音符／簽名四種圖形個別測試，最大誤差落在 **1.5×10⁻¹¹ px 至 2.2×10⁻¹¹ px** 之間，遠低於題目要求的 1×10⁻⁹ px 門檻（浮點數雙精度極限）。
2. **圓的數量越多、重建誤差是否單調下降**：對同樣四種圖形，取 M = 1, 3, 8, 15, 30, 50, 80, 120, 149, 150 逐一檢查，均方根誤差全數單調不增，例如愛心的誤差序列是 120.78px → 23.23px → 6.08px → 2.17px → 0.77px → 0.38px → 0.18px → 0.01px → 0.00px → 0.00px（400×400 畫布尺度下）；另外也證明並交叉驗證了上面提到的 Parseval 捷徑公式與暴力重算結果一致（誤差 < 10⁻⁹）。
3. **已知簡單訊號驗證頻率**：輸入一個半徑 60、頻率 f 的純圓形訊號，DFT 算出來振幅最大的係數，頻率精確等於 f、振幅精確等於 60（誤差 < 10⁻⁹），另外也測過雙頻率疊加訊號，兩個頻率的振幅與初始相位都被精確拆解回來。
4. 額外確認：頁面不含任何 `fetch`／`XMLHttpRequest`／外部 `<link>`／CDN 腳本／`@font-face`；HTML 各類標籤開合數一致；JS 大括號／小括號／中括號巢狀深度歸零（沒有漏配對）。
