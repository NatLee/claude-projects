# 狀態空間 · 一道光掃過序列（Mamba / SSM）

一個把「狀態空間模型（State Space Models, SSM）」與 **Mamba** 的核心直覺變成可以親手把玩的互動頁面。
一道光由左而右掃過一串 token，途中只維護**一個固定大小的隱藏狀態**；你可以點 token 叫模型「記住」它、拖遺忘閘看它記多久，再對比 Attention 的 O(N²)。

---

## 這是什麼

大型語言模型幾乎都建立在 Transformer 的 **self-attention** 上：每輸出一個字，都要「回頭看」序列裡的所有字。這很強，但成本隨長度**平方成長**，而且要為每個 token 快取 K/V（KV cache）。

**狀態空間模型**換了個思路——像訊號處理裡的濾波器，用一條遞迴式往前掃：

```
h_t = A·h_{t-1} + B·x_t      （更新固定大小的隱藏狀態）
y_t = C·h_t                   （從狀態讀出輸出）
```

它只帶著一個**固定大小的狀態**往前走，所以是**線性時間、不需要 KV cache**。
2023 年的 **Mamba** 解決了傳統 SSM（如 S4）最大的弱點：**參數是靜態的、無法依內容推理**。Mamba 讓 **B、C 與離散化步長 Δ 隨當前輸入變化**（A 也透過 Δ 一起被調變），於是模型能**「選擇」**要記住還是遺忘當前這個 token——這就是「**選擇性狀態空間（selective SSM）**」。

## 怎麼玩

1. **按「▶ 播放」**：一道光掃過每個 token，下方的隱藏狀態向量（一排長條）即時漲落。
2. **點任一個 token**（出現金色圖釘）＝ 標記為「要記住」。掃到它時，**金色的記憶通道**會把它「抓住」，並沿著後續 token 一路帶下去（token 下方的金色底線就是被帶著走的區間）。這就是 Mamba「選擇性複製」的直覺。
3. **遺忘閘 Δ 滑桿**：往右調（Δ 變大）→ 保留率 `a = e^(−Δ)` 變小 → 狀態衰減快、**記不久**；往左調 → **記很久**。可以看到藍色的上下文長條被明顯影響。
4. **序列長度 L 滑桿**（第二區）：拖動時，Attention 的配對次數（∝ L²）飆升、SSM 的更新次數（∝ L）幾乎貼著地面，兩條曲線把「長上下文為什麼貴」畫得一清二楚。

小提醒：畫面上「記憶通道」一次只抓得住**一個**被標記的 token（再抓一個就換人）——這剛好對應到 Mamba 真正的弱點：**把上下文壓進固定狀態，在「精確複製長序列」這類任務上較吃力**。

## 為什麼有趣

- 把一條看起來很抽象的遞迴式 `h ← A·h + B·x`，變成**看得見的一道光 + 一排會呼吸的長條**。
- 「選擇性」不是口號：你**親手**標記一個 token，就看到狀態把它挑出來、帶下去。
- 用同一頁把 SSM 與 Attention 的**複雜度差距**擺在一起，比背 O(N) / O(N²) 更有感。

## 方法論與誠實聲明（頁面是教學簡化版）

本頁的目標是「直覺」而非「重現論文」。核心遞迴的數學與真實 SSM 一致，但做了簡化：

- **離散化**：取穩定極點 `A = −1`，離散化 `Ā = exp(Δ·A)` 得保留率 `a = e^(−Δ) ∈ (0,1)`；零階保持（ZOH）下的輸入增益為 `B̄ = (1 − a)·B`。因此上下文維度的更新式其實是**輸入相依的指數移動平均**：`h ← a·h + (1−a)·Bx`——這正是「Mamba 式閘控」的直覺形式。
- **記憶通道**：這是為了讓「選擇性複製（抓住 → 保留）」看得見而設的 UI 機制。它示範了選擇性 = **依輸入決定 Δ**（用大 Δ 抓住一個 token、用小 Δ 保留它）；也順帶示範了「純量狀態一次只記得住一個」的限制。
- **真實 Mamba** 用的是**多維**狀態、結構化／對角的 A，以及**硬體感知的平行掃描（parallel scan）**；本頁用逐格遞迴呈現，數學核心相同（`h ← A·h + B·x`、`y = C·h`）。
- token 對應的向量以確定性雜湊（FNV-1a）產生，只是為了讓長條有穩定、豐富的動態，不代表任何真實嵌入。

### 驗證（node）

- **語法**：抽出頁面內兩段內嵌 `<script>`（純核心 + 互動）分別跑 `node --check`，皆通過。
- **核心遞迴（純函式，27 條斷言全過）**：
  - `ssmStep` / `ssmOutput` / `ssmScan` 對照手算（例：`a=0.5, b=1, xs=[1,0,0,0] → h=[1,0.5,0.25,0.125]`）。
  - 遺忘性質：`0<a<1` 時脈衝輸入後狀態 `h_t = a^t` **單調遞減、長程趨近 0**；`a` 越大同一步殘留越多（記越久）。
  - 線性掃描結果與「逐步展開」一致，且支援**逐步變化的參數**（即選擇性 SSM）。
  - 離散化 `Ā = exp(Δ·A)` 對 Δ 單調遞減、落在 (0,1)；輸入增益 `1 − a`。
- **執行期（jsdom 冒煙測試）**：載入頁面並實際觸發播放、單步走過被標記 token、重設、切換標記、三支滑桿、切換例句、`visibilitychange`、`resize`，全程無執行期錯誤。

## 技術備註

- 純靜態、單一檔案、零外部依賴，**雙擊 `file://` 即可開啟**（不使用 `fetch`、無 CDN）。
- 動畫只用 `transform` / `opacity`；掃描以 rAF 節流的計時器逐格推進，**分頁隱藏（visibilitychange）時自動暫停**。
- 完整支援 `prefers-reduced-motion`（用 `matchMedia` 的 `change` 事件動態監聽）：關閉自動掃描與畫線動畫，播放鈕改為「跑到底」直接顯示結果，仍可用「單步」逐格檢視。
- 無障礙：鍵盤可操作、聚焦樣式、`aria-label`、動態播報用 `aria-live`。
- **localStorage**：只存兩個偏好設定，皆加前綴 `ssm.`
  - `ssm.delta`：遺忘閘 Δ 的數值。
  - `ssm.speed`：掃描速度（1–5）。
