# 滾下山的球 · 梯度下降

神經網路「學習」的核心其實只有一件事：把誤差看成一座山，讓參數這顆球滾到谷底。這頁讓你放一顆球下去，親手玩學習率、動量與 Adam，看它收斂、彈飛、或卡在半山腰。

## 怎麼玩

1. 在曲線上**點一下**放球，或按「🎲 放球」。按 **▶ 自動下降** 看它一步步往低處滾；**單步** 可一步一步看。
2. 拉「學習率」滑桿：太小走不動、太大會彈飛。在「單谷（碗）」把學習率拉過 2，球會越滾越高、發散。
3. 切到「雙谷（有陷阱）」，用**梯度下降**會卡在右邊較淺的局部谷；換成**＋動量**，帶著慣性常能衝過中間的小坡、掉進左邊更深的全域谷。
4. 換 **Adam** 看它又快又穩地收斂。

## 為什麼有趣

一個有幾千億參數的大模型，訓練時就是在一座幾千億維的「誤差山」裡，靠「往最陡的下坡走一小步」這個動作重複幾兆次，慢慢滾向谷底。理解這顆球怎麼滾，就理解了深度學習的引擎。

三件反直覺的事，這頁都摸得到：

- **學習率是把雙面刃**：步伐太小，學一輩子也到不了；步伐太大，球越過谷底衝到對面更高處，來回彈跳甚至發散。對 `f(x)=½x²` 這個碗，學習率＝2 剛好是等幅震盪的臨界點，超過就爆走。
- **局部最小的陷阱**：山谷不只一個。純梯度下降滾進第一個坑就停了，哪怕旁邊有更深的谷。**動量**給球一點慣性，讓它有機會衝過小山坡、找到更低處。
- **Adam 幾乎是現代標配**：它替每個參數各自估計合適的步伐（用梯度的一階與二階動量），對學習率沒那麼敏感，這也是為什麼真實訓練絕大多數用它。

## 計算方法

- **地景**：單谷 `f(x)=½x²`（梯度 `x`）；雙谷 `f(x)=0.05(x²−4)² + 0.6x + 1.6`（梯度 `0.2x(x²−4)+0.6`），`0.6x` 的傾斜讓左谷（x≈−2.3）成為全域最小、右谷（x≈1.3）成為局部陷阱。
- **梯度下降**：`x ← x − lr·g`。
- **動量**：`v ← 0.9·v − lr·g；x ← x + v`（帶慣性）。
- **Adam**：一階動量 `m`、二階動量 `s`（β₁=0.9、β₂=0.999），各自偏差校正後 `x ← x − lr·m̂/(√ŝ+ε)`。

核心（`lossOf`、`gradOf`、`stepOpt`）都是純函式，通過 9 條 node 斷言：解析梯度＝數值梯度、碗小學習率收斂/大學習率發散/lr=2 等幅震盪、雙谷全域最小在左、純 GD 從右側卡局部谷、動量從右側能逃到全域谷、Adam 收斂。

## 技術

純靜態單檔，零外部依賴。曲線、軌跡與球用 canvas 繪製，`requestAnimationFrame` 驅動並在分頁隱藏時停止；`prefers-reduced-motion` 下改為一次跑完數十步、不做逐幀動畫，並以 `matchMedia` 的 change 事件動態監聽。canvas 依 `getBoundingClientRect()×dpr` 設定背景尺寸且只在尺寸改變時重設。本頁未使用 localStorage。
