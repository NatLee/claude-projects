# 生日悖論 · 23 個人的房間 🎂

把機率論裡最有名的反直覺結果變成一間可以親手擠人的房間：滑一滑人數，看撞生日的人當場亮起、連成一線；再讓電腦跑一萬次，證明這不是嘴上說說。

## 這是什麼？

「一個房間裡要幾個人，才有**超過一半**的機率，出現兩個人同月同日生？」

大多數人的直覺會喊出一兩百——正解卻是 **23**。只要 23 個人，就有約 **50.7%** 的機率至少兩人同一天生日。這個頁面讓你不用死背公式，直接「看」到、也「測」到它。

## 怎麼玩？

1. **先賭一把**：別急著往下拉，憑直覺挑一個數字（23／41／50／100／180／366）。選完會揭曉答案，並針對你的猜測給一句短評；按「滑到 23，給我看 →」直接跳到舞台。
2. **拉動人數滑桿（1–70 人）**：房間會即時長出人來，每個人頭上帶著一個生日。撞到同一天的人會亮成同一種顏色、頭頂冒出小點、彼此**連成一條發光的線**，並灑出彩帶慶祝。滑桿上標了 **23＝過半**的位置。
3. **看上方兩個大數字賽跑**：左邊「理論機率」隨滑桿即時滾動；右邊「實測頻率」等你按下模擬才登場。
4. **🎲 再擲一次**：人數不變、重新隨機抽一批生日。在 23 人時多按幾次，你會親眼看到它大約一半機率撞、一半機率不撞——這正是「50.7%」的手感。
5. **⚡ 跑一萬次**：電腦隨機開一萬間同樣人數的房間，數出真的撞到生日的比例，和理論值並排比較。你會看到實測值幾乎貼著理論值。

### 哇時刻

慢慢把滑桿往上推。理論機率一路爬，過了 **23 人**的瞬間跨過 50%、「已過半」的金色小徽章亮起；同一時間，房間裡第一次有兩個人亮成同色、一條光線把他們串起來、彩帶灑落——**「撞了！」**。原來這麼少人就夠了。

## 為什麼這麼少？

祕密全在「**任意兩人**」四個字。我們沒有指定是誰，只要湊得出任何一對同一天就算數。

- **配對數是平方暴衝**：23 個人能兩兩配出 **C(23,2) = 253 組**配對，每一組都是一次撞生日的機會。人數線性增加，機會卻以平方成長，機率因此衝得又急又快。
- **換成「跟你」同天就難了**：如果問的是「有沒有人跟**你**同一天」，那只剩 n 組配對，真的要約 **253 人**才過半。差別全在於：是任兩人，還是指定一人。
- **真的用在現實**：同一套數學就是密碼學裡的「**生日攻擊**」——想撞出雜湊碰撞，平均只需約 √N 次嘗試而非 N 次，這正是安全雜湊輸出必須夠長的原因。足球場上兩隊 22 人加一名主審剛好 23 人，也常被拿來當現成的驗證樣本。

## 數字正確嗎？

機率以 **365 天均勻分布**計算，忽略閏日 2/29，採標準公式：

```
P(至少兩人同天) = 1 − ∏_{k=0}^{n−1} (1 − k/365)
```

關鍵值皆經程式驗算：

| 人數 n | 理論機率 |
|:--:|:--:|
| 1 | 0% |
| 23 | **50.73%** |
| 41 | 90.32% |
| 57 | 99.01% |
| 70 | 99.92% |
| 366 | 100%（鴿籠原理：366 人「保證」至少兩人同天） |

蒙地卡羅以每批 10,000 次模擬，實測頻率與理論值誤差通常在 ±1% 內（例如 23 人跑一萬次約落在 50% 上下）。

**關於均勻假設**：真實世界的生日並非完全均勻，略有季節性分布。可以證明，在均勻假設下「過半所需人數」其實是**上界**——生日分布越不均勻，撞生日只會**更容易**，需要的人數只會更少或相同。所以 23 這個數字在現實中依然穩固。

## 設計與技術

- 純靜態單一 HTML，CSS／JS 全內嵌，離線雙擊 `file://` 即可開，**零外部依賴**（連彩帶、連線、曲線圖都是自己畫的）。
- 房間格子用 CSS Grid；撞生日的連線用一層 SVG 疊在格子上，漸層光暈以 `filterUnits="userSpaceOnUse"` 設定濾鏡區域，確保水平／垂直線也能正常發光；連線的畫入動畫用 dasharray 可靠重播。
- 動畫只用 `transform`／`opacity`，rAF 節流，分頁隱藏時暫停蒙地卡羅與數字滾動；完整支援 `prefers-reduced-motion`（開啟時不做逐格動畫與彩帶，直接呈現結果與數字），並動態監聽偏好變化。
- 鍵盤可操作、互動元素有 `aria-label`、動態結果用 `aria-live` 播報。
- **不使用 localStorage**，沒有任何需要記住的個人資料。
