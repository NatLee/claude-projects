# 疊塔 · 對齊、疊高、別手滑

一款純手感的街機小遊戲。方塊在頂端左右滑動，你抓準時機放下——和下面那塊**重疊的部分留下、超出的被切掉**，於是塔越疊越窄。塔越高，方塊滑得越快。目標很單純：疊到最高。

## 怎麼玩

- **放下方塊**：點畫面、輕觸，或按空白鍵／Enter。
- 每放一塊，和下方對齊的部分保留，其餘會被切掉飛走；切到一點不剩就結束。
- **完美對齊**（幾乎不差）會亮起「完美 ×N」連擊，還會把塔**稍微加寬回來**，讓你有翻身的機會。
- 最佳高度會記在瀏覽器裡（localStorage，key 前綴 `stack.`）。

## 一點設計

- 塔越高、方塊滑得越快（速度隨層數上升並封頂），難度自然升溫。
- 相機會隨著塔長高慢慢往上帶，讓當前這塊永遠在順手的位置。
- 完美對齊除了連擊，還會回補一點寬度——這是讓高手能一路衝上去的關鍵手感。

## 技術重點

- 單一 `index.html`，純 HTML/CSS/JS，無外部相依，雙擊 `file://` 即可玩。
- 遊戲主體用 canvas 繪製；**特別避開已知的 canvas 尺寸地雷**：畫布尺寸由 CSS（`.stage` 的固定高度）決定，JS 只讀 `clientWidth/clientHeight × dpr` 回寫一次，並在 `resize` 時依比例重算，**不使用 ResizeObserver**，因此不會產生每幀翻倍的回饋迴圈。
- rAF 迴圈在**分頁隱藏時暫停**（`visibilitychange`）；`prefers-reduced-motion` 時關閉切割碎片與光暈等裝飾動態（滑動屬遊戲本體，保留），並動態監聽變更。
- 疊放判定抽成純函式 `placeBlock`（重疊區、左右切割、完美判定），是唯一資料來源。

## 測試

- 核心 `placeBlock` 用 node 斷言：精確／誤差內完美、左右偏切割、剛好相接與完全錯開判失手；再跑 4000 條決定性隨機疊塔序列，驗證「新塊寬度永不變大、且完全落在舊塊範圍內」的不變式，並確認偏移夠大時會正確判定失手。
- 用 linkedom 實際載入頁面（shim 掉 canvas 2D 與 rAF），確認能啟動、rAF 迴圈有在繪製、放下方塊會計分、最佳成績寫入 localStorage，且全程無執行錯誤。
- 測試腳本直接從本頁 HTML 抽出 `STACKCORE` 區塊，確保頁面內的核心與測試一致。全站另跑 `node tools/check.js` 通過。
